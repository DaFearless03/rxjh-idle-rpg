# Phase6 完整玩家旅程测试

目标：用一条主线流程验证 v1.0 从创角到 3 转毕业不会崩溃、丢档或卡流程。

## 测试前准备

- 使用 Chrome / Edge / Safari 16.4+ 打开 `v1.0/index.html` 或本地 http server。
- 测试前导出当前存档备份，或在 GM 面板确认可以恢复。
- 建议测试角色名：`测试刀客`，职业选择 `warrior_blade`。
- 可使用 GM 面板加速升级、给药剂、给盒子，但每个阶段完成后都要保存并刷新页面确认数据能恢复。

## 主流程

1. 清空测试环境或进入空槽位，创建 `warrior_blade` 角色 `测试刀客`。
2. 确认新角色：Lv1、HP/MP 为满、金币为初始值、位置在泫渤派城镇。
3. 进入 `xuanbo_field` 或同等级野外区，点击开始挂机。
4. 升到 Lv10，确认挂机过程无控制台错误，死亡后会回城并停止挂机。
5. 回城与泫渤派门主交互，接取 1 转任务 `quest_transfer_1`。
6. 挂机收集 `boar_leg` 和 `wolf_bone`，任务面板进度应同步增加。
7. 任务物品足够后回城提交 1 转任务，确认职业变为 `warrior_blade_transfer_1st`。
8. 打开背包，手动使用 HP/MP 药剂，确认数量扣除、HP/MP 增加、日志/提示正常。
9. 给或掉落至少 1 个盒子，点击背包盒子开盒，确认盒子扣除、奖励入背包、满包时有丢弃提示。
10. 装备一件基础装备，尝试强化和合成石头，确认属性刷新且保存后仍恢复。
11. 挂机或 GM 加速到 Lv35，接取 2 转正派任务 `quest_transfer_2_positive`。
12. 接取 2 转任务后确认 `faction` 变为 `positive`，继续收集任务物品并提交。
13. 提交后确认职业变为 `warrior_blade_transfer_2st_Z`，已完成任务写入 completed。
14. 配置自动喝药：选择 HP/MP 药剂和阈值，进入 35-60 区间野外挂机。
15. 长时间挂机期间确认自动喝药不清挂机，手动喝药、开盒、换装、强化、合成也不清挂机。
16. 升到 Lv60，接取 3 转正派任务 `quest_transfer_3_positive`。
17. 按任务阶段逐步收集并提交，确认阶段推进、任务物品扣除和 UI 进度一致。
18. 提交 3 转任务后确认职业变为 `warrior_blade_transfer_3st_Z`。
19. 保存、刷新页面，重新进入角色，确认职业、等级、背包、装备、气功、任务完成状态都恢复。
20. 开启挂机后关闭页面一段时间，再次进入，确认离线收益弹窗正常显示且不会卡死。

## 通过标准

- 全流程无启动错误、无未捕获异常、无页面卡死。
- 每次刷新后角色状态、背包、装备、任务、位置、挂机配置都能恢复。
- 手动喝药、自动喝药、开盒、离线收益都能正常与背包数量联动。
- 任务转职链路能到达 Lv60 + 3 转职业。
