# Phase6 边界场景检查清单

目标：集中验证 v1.0 上线前最容易导致坏档、卡死或玩家误解的边界。

## 当前验收记录

- 2026-06-05：配置校验、引用完整性、3 转多阶段提交、商店未配置物品防线、强化石 02 使用已自动覆盖。
- 仍建议人工验证：长时间挂机、真实离线 24h+、跨浏览器导入导出、localStorage 损坏恢复、移动端连续操作。

## 存档与启动

- 启动配置校验：故意把一个盒子 `openable_items[0].item_key` 改成不存在的 key，刷新后应显示启动错误并阻止进入游戏。
- 启动配置校验：故意把一个怪物 `drop_items.item_key` 改成不存在的任务物品，刷新后应显示配置校验失败。
- 脏导入恢复：设置 `localStorage.import_in_progress = "true"` 后刷新，应清理脏状态并回到创建角色流程。
- 跨设备导入：导出全部角色，在空浏览器导入，重启后角色列表、last_used_slot、背包、任务状态应一致。
- localStorage 异常：模拟存档 JSON 损坏，启动不应白屏，应进入现有恢复或创建流程。

## 背包与物品

- 手动喝药：HP 未满时使用 HP 药，数量扣除，实际恢复量等于 `base * (1 + healBonus)` 且不超过 maxHp。
- 手动喝药：MP 未满时使用 MP 药，数量扣除，实际恢复量等于 `base * (1 + mpRecoveryBonus)` 且不超过 maxMp。
- 手动喝药：等级不足使用高阶药，应 toast “等级不足”，背包数量不扣。
- 手动喝药：HP/MP 已满时使用对应药剂，应提示已满，背包数量不扣。
- 药剂堆叠：同 key 药剂单格最多 999，超过后新开格，不应被 99 上限截断。
- 开盒：点击盒子默认数量为当前拥有数量，确认后按数量扣盒子。
- 开盒：奖励进入背包，装备奖励创建 equipment instance，药剂/石头奖励走堆叠。
- 开盒满包：背包满时可丢弃非任务奖励并提示丢弃数量，不应崩溃。
- 任务物品：任务物品不可手动使用，背包点击应只提示不可使用或出售。

## 战斗与挂机

- 玩家死亡：任务进度保留、装备保留、HP/MP 回满、`is_auto_play=false`。
- 手动喝药不清挂机：挂机中打开背包使用药剂，挂机状态保持。
- 开盒不清挂机：挂机中打开背包开盒，挂机状态保持。
- 自动喝药：药剂耗尽时不降级、不停挂机、不弹阻塞弹窗。
- 自动回城补给：金币不足且补给后仍触发阈值时停止挂机并给出提示。
- 切换 zone：玩家主动切换野外/城镇应清挂机，自动回城补给和死亡复活不重复清挂机。

## 离线收益

- 离线 24h+：结算截断至 24h，有提示，不应卡死。
- 离线死亡：`stopped_reason="death"`，`died_at_s` 有值，`is_auto_play=false`。
- 离线任务物品满包：`stopped_reason="inventory_full_quest_item"`，弹窗显示任务物品丢失详情。
- 离线获得盒子/装备/石头：summary 中统计盒子、获得物品和丢弃数量。

## UI 与运行稳定性

- ESC 只关闭最上层 modal，不应破坏主页面 active panel。
- Ctrl+Shift+G 只切换本地 GM 面板，线上缺失 `gm.js` 不应影响启动。
- 角色切换：`UIState.active_npc` 清空，NPC 弹窗关闭。
- 返回角色列表：当前角色先保存，角色列表展示 last_used_slot 和各槽位状态正确。
- 长时间挂机 1 小时：战斗日志和掉落日志维持 200 条环形缓冲，滚动监听不重复绑定。
- 页面隐藏/失焦：触发保存节流，不应频繁写盘或报错。
