# Phase6 验收结论

验收日期：2026-06-05

## 结论

Phase6 当前达到可进入下一阶段的收尾标准：

- 启动配置校验为 `0 errors / 0 warnings`。
- 任务怪物引用、怪物任务掉落、地图装备/石头掉落池、NPC 商店物品、转职奖励职业均无悬空引用。
- 4 个基础职业（刀客、剑客、枪客、医师）均可创建，并拥有 1/2/3 转职业链配置。
- 3 转多阶段任务已覆盖“部分阶段不可提交”和“全阶段可提交并按职业系转职”两类路径。
- v1.0 UI 商店不再展示未上线的弓手物品，UI 购买入口会拒绝未配置物品。
- 浏览器启动 smoke 通过，控制台无 error/warning。

## 已闭环问题

1. `drop_monster` 悬空引用：补齐 3 转任务怪物、任务掉落和对应地图刷怪池。
2. 3 转正派嗜血狂狼掉落不完整：补齐皮、爪、尾毛、血。
3. `equipment_pool` / `stone_pool` 悬空引用：补齐二档装备、二档石头，并兼容 `enhance_stone_02`。
4. `shop_item` 悬空引用：补齐 NPC 商店出售的基础多职业武器、防具、手套、靴子模板。
5. `reward_career` 悬空引用：补齐剑客、枪客、医师的基础和 1/2/3 转职业链。
6. UI 商店弓手泄漏：移除 v1.0 未上线的弓手商品。
7. 多阶段任务误提交：提交前补跑阶段推进，只有所有阶段完成才允许结算奖励。
8. 装备元数据缺口：补充装备 `itemMeta`，避免背包展示只露 key。
