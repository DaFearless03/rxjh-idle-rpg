挂机游戏开发文档

本文档为文字类挂机游戏的标准开发文档，可直接用于AI读取并执行编码。
修改配置区块中的数值和规则即可定制新游戏。

文档约定（v1.0）：
- 所有 YAML 配置中以 : | 开头的多行字符串（如 hit_check: | / evaluation_flow: | / gain_per_level: |）
  都是 Python 风格伪代码（用于描述算法逻辑，便于人读）。
  实现时按 ES6/TypeScript 翻译，注意以下常见映射：
    - Python x if cond else y        → JS cond ? x : y
    - Python for i in range(n)       → JS for (let i = 0; i < n; i++)
    - Python dict.get(k, default)    → JS obj[k] ?? default
    - Python len(arr)                → JS arr.length
    - Python arr.append(x)           → JS arr.push(x)
    - Python floor(x)                → JS Math.floor(x)
    - Python random() / random.random() → JS Math.random()
    - Python min/max(a, b)           → JS Math.min/max(a, b)
- 中文注释（# ...）是字段说明，不参与运行时逻辑
- 数据字段（数字 / 字符串 / 数组 / 映射）严格按 YAML 标准解析
v1.0版本说明：
- 等级上限：60级
- 可选职业：刀客系、剑客系、枪客系、医师系（弓手系、刺客系不上线）
- 开放地图：泫渤派、柳正关、三邪关、神武门、柳善提督府、南明湖（南明湖为高等级探索区 L68-75，不锁入口；v1.0 等级上限 60 的玩家可进入但难以战胜）
战斗设计取舍（v1.0）：
- 1vN 战斗模型：同 zone 同时最多 8 只怪独立攻击玩家，怪 atk 不下调
- 玩家承担"打同级 N 只怪可能秒杀"的风险，需通过【升级装备 / 强化合成】或【主动选择低 5-10 级 zone 刷怪】来调节节奏
- 玩家死亡损失 1% 当前级所需经验、不掉级、不掉装备、回出 zone 时所在城镇
- 转职任务"硬仗"取舍：任务接取等级（10/35/60）= 当前装备段位上限的"上一段"——即玩家接 2 转任务时只能穿 t1 装备打 L35-37 怪，3 转任务时只能穿 t2 装备打 L59-61 怪。
设计意图：转职任务是"成长里程碑"，要求玩家用当前段位最强装备（满 4 孔寒玉石 / 强化+6 / 满气功）"硬刚"高一档怪物；不是 bug，不要降任务怪等级或升装备段位。
- 装备永久无耐久度（v1.0 不做修理/损坏系统）；金币消耗靠强化、合成、药剂购买三大通道，详见 9.1 economy.expenses
- 详见 6.1 battle_flow.battle_model / player_death
